# My Thoughts - 2021

## Year Theme: Architecture & Foundation Building

2021년은 백엔드 개발자로서 **소프트웨어 아키텍처**에 깊이 빠진 해였다. 단순한 기능 구현을 넘어 시스템 전체를 설계하고 구조화하는 능력을 키우는 데 집중했다.

## Key Learnings & Growth

### 1. 백엔드 집중의 가치

**좋았던 점**:
- 백엔드에 집중할 수 있었음
- 다양한 시도를 해볼 수 있어서 소프트웨어 아키텍처 개선에 도움이 됨

**깨달음**:
- 전문성의 중요성: 여러 분야를 얕게 아는 것보다 한 분야를 깊이 파는 것이 더 가치 있음
- 백엔드 엔지니어로서의 정체성 확립

### 2. 대규모 프로젝트 경험

#### 수자원공사 프로젝트 (난이도 ⭐⭐⭐⭐⭐)
**기술적 성장**:
- 도커를 조금 더 자유롭게 사용할 수 있게 됨
- gRPC를 도입하여 아키텍처를 구성할 수 있게 됨
- **실제로 대규모 트래픽을 받아본 첫 번째 프로젝트**

**가장 중요한 학습**:
- 36대 CCTV 동시 처리라는 실제 대규모 트래픽 경험
- 이론으로만 알던 스케일링이 실제로 어떤 문제를 일으키는지 체험
- 시스템 부하 문제를 납품일 내에 해결하며 압박 상황 대처 능력 향상

#### DNA+ 프로젝트 (난이도 ⭐⭐⭐⭐)
**새로운 패러다임**:
- **DDD(Domain-Driven Design)를 적용하여 프로그래밍**
- 단순 코딩이 아닌 도메인 모델링의 중요성 깨달음
- 대규모 프로젝트의 문서 작성 경험

**기술 스택 확장**:
- RTSP, KLV, MJPEG 방식의 다양한 스트리밍 프로토콜 습득
- 실시간 데이터 처리에 대한 이해 증진

### 3. 문서의 힘

#### 인천TP 프로젝트 (난이도 ⭐)
**단순하지만 중요한 깨달음**:
- **문서만으로 API 연동이 가능하다는 것을 깨달음**

**반성과 학습**:
- 좋은 문서는 커뮤니케이션 비용을 획기적으로 줄인다
- API 설계 시 문서화의 중요성
- 내가 만든 시스템도 타인이 문서만으로 사용할 수 있어야 함

### 4. 스트리밍 기술 마스터

#### 승강기 안전공단 프로젝트 (난이도 ⭐⭐⭐)
**기술적 획득**:
1. **FFmpeg 스트리밍**을 사용할 수 있게 됨
2. **소프트웨어 장애 대처**를 고려하게 됨
3. **Pipe 통신**을 사용할 수 있게 됨

**소프트웨어 안정성에 대한 인식**:
- 기능이 동작하는 것과 안정적으로 동작하는 것은 다르다
- 장애 시나리오를 미리 고려하는 습관의 중요성
- Fail-safe 설계의 필요성

### 5. 인프라 경험

#### AI Hub 서버 셋팅
**숙련도 향상**:
- 이전: 서버 셋팅에 일주일 소요
- 이후: 3일로 단축

**실무 경험**:
- 실제 서버실에서 물리적 서버 셋팅 경험
- 이론과 실제의 차이 체험
- 트러블슈팅 역량 향상

#### NAS 서버 구축 (단독 프로젝트)
**사용자 중심 사고**:
- 팀원의 작업 레벨을 고려하여 친숙한 Windows GUI 환경 선택
- 기술적 우월성보다 팀 생산성이 중요함을 깨달음
- 교육 및 온보딩의 중요성

**협업 인프라 구축**:
- 24TB NAS로 팀 협업 효율성 향상
- 네트워크 드라이브, RDP 설정으로 원격 작업 환경 구축
- 인프라가 팀 생산성에 미치는 영향 실감

## Self-Reflection & Critique

### 좋았던 점
1. **백엔드에 집중**: 전문성 확립
2. **다양한 시도**: 7개 프로젝트 경험
3. **기술 확장**: API 수 4개 → 11개
4. **지식 공유**: 17개 기술 글 작성, 11개 논문 리뷰

### 나빴던 점
1. **요구사항만 겨우 충족**: 그 이상을 적용하지 못했음
2. **표준을 따르지 못한 프로젝트 다수**: 기술 부채 발생

**반성**:
- 납기일에 쫓겨 표준과 베스트 프랙티스를 무시한 경우가 많았음
- 단기적으로는 성공이지만 장기적으로는 유지보수 비용 증가
- "빠르게 만드는 것"과 "제대로 만드는 것"의 균형 필요

### 지키지 못한 목표
1. 2주에 1개씩 미니 프로젝트 설계 및 리뷰
2. 매주 Stack Share에서 프로젝트 설계 리뷰 글 작성하기

**원인 분석**:
- 회사 프로젝트 일정에 치여 개인 학습 시간 확보 실패
- 목표 설정은 했으나 우선순위 관리 미흡
- 지속 가능한 목표 설정 필요

## Deep Dive: Architecture & Design

### 소프트웨어 아키텍처 탐구

**주요 관심사**:
- 소프트웨어 아키텍처
- 마이크로서비스 아키텍처
- 프로젝트 기획
- 프로그래밍 기법

**학습 활동**:
- GoF 디자인 패턴 정독 (훈련소에서)
- 도메인 주도 설계 철저 입문 독서
- 클라우드 네이티브 애플리케이션 가이드라인 학습

**실전 적용**:

**학습한 패턴들**:
1. **Request-Reply 패턴**: 비동기 통신
2. **Pub-Sub 패턴**: 이벤트 기반 아키텍처
3. **Eventual Consistency**: 분산 시스템 일관성

**실무 적용 경험**:
- gRPC 버전 ReCon_API 설계
- 동적 스케일링 가능한 아키텍처 설계
- 모니터링 시스템 구축 (Fluentd + InfluxDB)

## Technical Identity Formation

**Before 2021**:
- 다양한 것을 조금씩 아는 제너럴리스트 지향

**After 2021**:
- 백엔드 전문가로서의 정체성 확립
- 아키텍처 설계 역량에 집중
- 시스템의 안정성과 확장성을 책임지는 역할

### 아키텍트로의 여정

**현재 위치**:
- 단순 기능 구현 → 시스템 설계
- 코더 → 엔지니어
- 실행자 → 설계자

**목표 방향**:
- 소프트웨어 아키텍트로의 성장
- 기술적 의사결정 능력 향상

## Final Reflections

### 종합 평가

**성장 지표**:
- 프로젝트: 5개 → 7개
- API: 4개 → 11개
- 기술 글: 17개
- 논문 리뷰: 11개
- 독서: 3권

**핵심 성과**:
- 대규모 트래픽 처리 경험
- gRPC, DDD 등 현대적 아키텍처 패턴 습득
- 백엔드 전문가로서의 정체성 확립

**앞으로의 다짐**:
- 요구사항만 충족하는 것이 아닌, 표준을 지키는 개발
- 빠른 개발과 올바른 개발의 균형
- 지속 가능한 학습 습관 확립
- 협업과 지식 공유 문화 선도

### Key Mindset Shifts

1. **기능 구현** → **시스템 설계**
2. **빠르게** → **제대로**
3. **혼자** → **함께**
4. **코드** → **아키텍처**
5. **실행** → **설계와 실행**

2021년은 백엔드 엔지니어로서의 기반을 다진 한 해였다. 2022년에는 이 기반 위에 DevOps와 협업 문화를 쌓아 더 성숙한 엔지니어로 성장하고 싶다.
