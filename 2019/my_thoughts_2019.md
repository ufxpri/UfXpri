# My Thoughts - 2019

## Year Reflection: The Beginning

2019년은 백엔드 엔지니어로서의 시작점이었다. 하지만 2020년 초의 회고를 보면, 이 해는 "그냥 살았던" 해였다. 성장보다는 결과에 집중했고, 능동적이기보다는 수동적이었다.

## The Way I Worked in 2019

### 결과 중심의 일
- 성장을 중심으로 하지 않고 **결과를 중심으로 일**했다
- 수치로는 성과가 보일지 몰라도 개인 성장에는 신경 쓰지 못했다
- 프로젝트를 완료하는 것이 목표의 전부였다

### 논문을 읽지 않았던 이유
**왜 읽지 않았나?**:
- 논문을 읽어야 할 필요가 없었다
- 내가 직접 모델 코드를 작성하는 일이 없었다
- 따라서 논문의 중요성을 알지 못했다

**무엇을 몰랐나?**:
- 논문에는 앞으로 무엇을 해야 할지 가이드를 줄 수 있는 힌트가 있다는 것
- 새로운 기술을 배우는 원천이라는 것
- 전문성을 키우는 방법이라는 것

### 선배 의존의 나태함
- 선배가 너무 잘하고 있었다
- 가만히 있어도 신기술을 적용한 모델이 쭉쭉 나왔다
- 딱히 내가 신경 쓸 필요가 없었다

**결과**:
- 나는 개발에만 집중했다
- 회사가 원하는 방향과는 조금 맞지 않게 되었다
- 수동적인 자세로 일했다

## What I Actually Accomplished

### AI API 프로젝트

#### Technical Achievement
- 3개 이상 업체에 동시 배포
- REST API 설계 및 구현
- JWT 인증 시스템 구축
- 비동기 처리 구현
- 배치 GPU 스케줄링

#### 의미 있었던 것
**성과는 있었다**:
- 여러 업체에 실제 서비스 제공
- 안정적인 운영
- 성능 최적화

**하지만 아쉬움**:
- 단순히 기능을 구현하는 데 집중
- 왜 그렇게 설계했는지에 대한 깊은 고민 부족
- 더 나은 방법에 대한 탐구 부족

## Key Learnings (Realized in Retrospect)

### 1. 사용자 중심 설계의 중요성

#### 각 업체별 맞춤형 UI
- 업체의 특징을 고려한 UI 설계
- 클라이언트 요구사항 반영

**깨달음 (나중에)**:
- 사용자 중심 설계의 중요성
- 같은 API라도 UI가 다를 수 있다
- 클라이언트마다 다른 니즈가 있다

### 2. 비동기 처리의 필요성

#### 분석 단계 추적 가능한 인터페이스
- 비동기 방식 API 설계
- 진행 상황 모니터링

**의미 (나중에 깨달음)**:
- 사용자는 진행 상황을 알고 싶어 한다
- 긴 작업은 비동기로 처리해야 한다
- 추적 가능성이 사용자 경험을 높인다

### 3. 리소스 최적화

#### 배치 단위 GPU 스케줄링
- 효율적인 GPU 사용
- 리소스 최적화

**배운 것 (암묵적으로)**:
- GPU는 비싼 자원이다
- 배치 처리가 효율적이다
- 스케줄링이 중요하다

## What I Missed in 2019

### 1. 능동적 학습 부재
- 논문을 읽지 않았다
- 새로운 기술을 직접 찾아보지 않았다
- 선배가 알려주길 기다렸다

### 2. 성장 중심 사고 부족
- 결과만 중요했다
- 과정에서 배우는 것에 무관심
- KPI는 있었지만 개인 성장 목표는 없었다

### 3. 깊이 있는 이해 부족
- 왜 이렇게 설계했는지 고민 부족
- 더 나은 방법 탐구 안 함
- 동작하면 만족

### 4. 기술적 호기심 부족
- 내가 사용하는 기술의 내부 동작 무관심
- Flask가 어떻게 동작하는지?
- JWT가 왜 안전한지?
- 배치 처리가 왜 효율적인지?

## The Contrast: 2019 vs 2020

### 2019 (이 해)
- 결과 중심
- 수동적 학습
- 선배 의존
- 논문 안 읽음
- 개발만 집중
- 회사 방향과 어긋남

### 2020 (다음 해)
- 성장 중심
- 능동적 학습
- 논문 읽기 시작
- 클라우드, 도커 도입
- 회사와 방향 맞추기

## Self-Assessment (Honest)

### 좋았던 점
- **실제 서비스 경험**: 3개 업체 배포
- **기술 습득**: Flask, JWT, 비동기 처리
- **문제 해결**: GPU 스케줄링, 배치 처리

### 아쉬운 점
- **수동적 태도**: 시키는 일만 함
- **성장 무관심**: 결과만 중요
- **학습 부족**: 논문 안 읽음, 깊이 없음
- **방향성 부재**: 회사와 내가 다른 길

## Critical Reflections

### "그냥 살았다"
2020년 초의 회고에서 "2019년은 그냥 살았던 것 같다"고 했다. 이 표현이 2019년을 잘 요약한다.

**의미**:
- 목적 없이 일했다
- 성장 없이 시간만 보냈다
- 주도적이지 못했다

### 선배에게 의존
"선배가 너무 잘해서 나태했다"는 고백은 중요하다.

**문제**:
- 외부 의존
- 주도성 상실
- 편안함에 안주

**결과**:
- 개인 성장 정체
- 회사 방향과 불일치
- 2020년의 각성 필요

## What This Year Taught Me (Later)

### 실전 경험의 가치
- 3개 업체 배포는 귀중한 경험
- 실제 클라이언트 대응 경험
- 운영 및 유지보수 경험

### 기술 스택 기초
- Flask API 개발
- JWT 인증
- 비동기 처리
- GPU 최적화

**하지만**:
- 깊이가 부족했다
- 왜?를 묻지 않았다
- 더 나은 방법을 찾지 않았다

## The Turning Point (Looking Forward)

2019년은 "그냥 산" 해였지만, 이것이 2020년의 각성을 만들었다.

**2020년의 변화 예고**:
- 논문 읽기 시작
- 능동적 학습
- 성장 중심 사고
- 클라우드, 도커 도입

**2019년의 의미**:
- 실전 경험 축적
- 문제점 인식의 토대
- 변화의 필요성 체감

## Final Thoughts

2019년은 성장의 해는 아니었지만, **변화의 씨앗이 뿌려진 해**였다.

"그냥 살았다"는 자각이 2020년의 "성장 시작의 기점"을 만들었다. 수동적이었고, 선배에게 의존했고, 논문을 읽지 않았던 이 해가 역설적으로 2020년의 각성을 준비했다.

**핵심 교훈 (나중에 깨달은 것)**:
1. 결과만으로는 부족하다 - 성장이 중요
2. 수동적으로는 성장할 수 없다 - 능동적 학습 필요
3. 편안함은 위험하다 - 항상 도전해야
4. 의존은 나태함을 만든다 - 스스로 찾아야
5. 논문은 미래를 보여준다 - 읽어야 한다

2019년, "그냥 살았던" 이 해가 2020년의 변화를 위한 반면교사가 되었다.
