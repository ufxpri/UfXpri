## 🗓️ 프로젝트별 진행 과정

### 🌊 소양강댐 영상안전관리시스템 (수도시설 지능형 영상 안전관리/침전지 물관리)
**기간**: 2024.02 ~

*   **배경 및 문제**
    *   기존 Classification 모델의 데이터 편향으로 인한 오검출(의자, 나무 등) 및 미검출 문제 발생.
    *   초기 배포 시 환경부 점검 대비 안정성 및 검출 성능 확보 필요.
    *   높은 시스템 리소스 사용량(CPU 330% 등)으로 인한 최적화 필요성 대두.
    *   운영 서버 담당사 변경(이화트론 → 네이즈)으로 인한 개발 스펙 및 아키텍처 변경 발생.
    *   현장 인터넷망 제약으로 인한 오프라인 배포 및 업데이트 환경 구축 필요.
    *   **[New]** 분석 지연(Delay) 및 대용량 패치 파일로 인한 유지보수 비효율성 발생.
    *   **[New]** 다수 지사(정수장) 원격 관리의 어려움 및 현장 네트워크 대역폭 제한 (400KB/s 수준).
    *   **[New]** CCTV 동시 추가 시 연결 실패 및 쓰레기 값 출력으로 인한 분석 중단 이슈 발생.

*   **해결 과정 (Timeline)**
    *   **2024-02-06**: 검출 모델을 CLIP 모델로 교체하여 정확도 향상 시도. GPU 사용량이 37%에서 85%로 증가했지만, 오검출 감소 효과 확인. "채널 수" 산정 기준을 재수립하여 안정적인 운영 범위 도출.
    *   **2024-02-21**: 특성 유사 객체(사람 간 구분) 및 비사람 객체(의자, 나무) 오검출 해결. OMP 파라미터 튜닝을 통해 CLIP 모델 CPU 사용량을 330% → 160%로 약 50% 절감. 관제 지연 이슈 트러블슈팅.
    *   **2024-02-28**: `ai-module` 리팩터링 완료 및 배포. 안정성 테스트 중 `ConnectionError` 및 네트워크 I/O 스파이크 현상 발견 및 분석.
    *   **2024-03-05**: 현장 설치를 위한 운영체제 및 SW 설치 가이드 문서화. 로깅 시스템(Loki/Grafana)을 안정성을 위해 로컬 설치 방식으로 결정.
    *   **2024-03-11 ~ 03-18**:
        *   운영 서버 담당사 변경에 따라 이벤트 연동 규격을 **Webhook 방식에서 TCP/IP 소켓 통신**으로 변경하고 인터페이스 재설계.
        *   기존 운영사가 담당하던 **SCADA 시스템 연동** 및 **알고리즘 설정(AI-Config)** 기능을 OGQ가 이관받아 개발 착수.
        *   알고리즘 민감도(Threshold) 설정을 위해 0~1 사이의 10단계 **Preset** 개념 도입 (UI/DB 변경 최소화).
    *   **2024-03-26**: 사내 운영/분석 서버 환경 구성 및 통합 검출 테스트 성공. Admin UI 개선 (기본값 할당 등).
    *   **2024-04-02 ~ 04-09**:
        *   **AI 모듈 리팩터링**: `Worker` 추상 클래스 및 `InferWorker` 도입으로 코드 구조 단순화 (AI 팀 코드 적용 시간 1일 단축).
        *   **TCP 서버 리팩터링**: 이벤트 전송 모듈 구조 개선.
        *   **기능 개발**: 사람(Person) 검출/세그멘테이션 모델 적용, 영역 설정(ROI) 반응형 UI 개발, 이벤트 뷰어 원본/이벤트 스트림 URL (`mjpeg`) 추가.
    *   **2024-04-17 ~ 04-30**:
        *   **오프라인 배포 체계 구축**: 인터넷 없는 현장 환경을 고려하여 Docker 이미지를 **Tar 파일(약 35GB)**로 빌드하여 배포하는 프로세스 수립 및 테스트 (`docker save/load`).
        *   **현장 지원**: 성남/구미 정수장 방문 설치 및 데모 시연. CCTV 접속 문제(ID/PW, Ping 실패) 및 SCADA 현장 이슈 트러블슈팅 진행. 누수 모델 업데이트.
    *   **2024-05-14 ~ 05-30**:
        *   **SCADA 연동 안정화**: 성남 정수장 현장 연동 테스트 성공(5/23). 태그 추가 및 설정 프로세스(신우산전 협업) 정립 및 검증 완료.
        *   **성능/안정성 최적화**: 다수 객체 검출 시 발생하는 분석 지연(최대 1분) 문제를 해결하기 위해 **"Drop 전략"**(과부하 시 프레임 스킵) 적용. 알고리즘 프로세스 분리를 통해 영상 단절 시 무한 대기 버그 근본 해결.
        *   **배포 효율화**: Docker 레이어 구조를 개선하여 소스코드와 베이스 이미지를 분리, 업데이트 패키지 용량을 **30GB → 12GB**로 약 60% 경량화 성공.
        *   **시스템 고도화**: Admin UI 일괄 수정/채널 제어 기능 추가. Naiz 연동 프로토콜 최적화(이미지 전송 시점 축소) 및 텍스트 시각화 구현.
    *   **2024-07-23**:
        *   **원격 통합 관리 도입**: **Ansible**을 도입하여 다수의 정수장 서버를 효율적으로 관리하는 체계 마련.
        *   **네트워크 최적화**: 현장의 낮은 대역폭(400KB/s)과 불안정한 연결에 대응하기 위해 `rsync`의 **partial transfer(이어받기)** 기능을 활용한 재전송 전략 수립.
        *   **SCADA 확장**: 잔여 6개 지사(덕소, 와부, 용인 등)에 대한 태그 연동 및 자동 추적 기능(Auto-tracking) 연계 계획 수립.
    *   **2024-08-06 ~ 08-27**:
        *   **안정성 확보**: CCTV 동시 등록 시 발생하는 연결 실패 및 데이터 오류 버그 수정 (우선순위 최상). 태백(프리징), 성남/충주(Black Screen) 현장 이슈 대응을 위한 데이터 수집 및 분석.

        *   **원격 업데이트 시스템 개발**: Docker Registry 구축 및 레이어 기반 업데이트 시스템 개발 완료 (8월 말 구축, 현장 적용은 11월로 연기).

        *   **액티브 러닝 구축**: Detection DB 구현 완료 및 데이터 관리(pickle/json) 설계. 레이블링 UI 기획 진행.

        *   **기능 추가 요청**: Naiz 측에서 이벤트 검증을 위한 **클립 백업(Clip Backup)** 기능 개발 요청 접수.
