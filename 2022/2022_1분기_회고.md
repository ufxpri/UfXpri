# 이번 분기의 교훈

- 모든 기능을 전부 제공하는 것이 항상 정답은 아니다.
- 초반에 설계한 사양은 반드시 바뀐다. 변화를 대비하여 설계 해야한다.
- python 은 인터프리터 싱글 쓰레드 언어이다. 기능을 설계할때에는 이러한 성질을 고려하여야 한다.

# 앞으로의 방향

기존 시스템에는 안정성이 부족하였음. 과거 경험을 기반으로 모니터링 계속 진행할거고 급한 일정때문에 다루지 못했던 부분들 보완할것임. 따라서 프로젝트를 빨리 끝내는것보다 안정적으로 설계하는것이 우선시 될 것
