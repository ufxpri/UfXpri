### DNA+

- 이번 분기 작업
  - REST API 형태로 시스템 다시 제작
  - 총 3번의 시연
- 다음 목표
  - 모델 구동속도 최적화
    - 모델이 느린 환경에서는 어떠한 방법을 사용하여도 최적화에 한계점이 있음. 너무나 당연한 사실 이지만 시스템 성능 개선 목표를 잡을 때 모델 실행 시간과 요구되는 스펙을 잘 비교하여 모델이 낼 수 있는 성능의 100%를 내고 있는지를 측정해야함
  - 이미지 스케쥴링 문제 해결
    - 두 프로세스 간의 병목현상 최소화를 위해선 두 시스템의 타이밍을 계산할 수 있어야 함. 각 시스템을 단편적으로 바라보고 설계를 진행할 시 예상치 못했던 곳에서 새로운 형태의 병목현상이 발생할 수 있음
