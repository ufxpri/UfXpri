# My Thoughts - 2020

## Year Theme: Awakening & Transformation

2020년은 **"성장 시작의 기점"**이었다. 결과 중심에서 성장 중심으로, 개인 작업에서 팀 협업으로, 레거시에서 현대 기술로의 전환을 경험한 한 해였다.

## Q1: The Awakening

### 깨달음의 순간

#### 동료들의 성장이 눈에 보였다
2019년에는 그냥 살았다. 성장을 중심으로 하지 않고 **결과를 중심으로 일**했다. 수치로는 성과가 보일지 몰라도 나 개인의 성장에는 신경 쓰지 못했다.

**구체적 사례**:
- 논문을 잘 읽지 않았다
- 이유: 논문을 읽어야 할 필요가 없었기 때문
- 내가 직접 모델 코드를 작성하는 일이 없었음
- 논문이 앞으로 무엇을 해야 할지 가이드를 줄 수 있는 힌트인 것을 몰랐음

#### 선배 의존의 나태함
선배가 너무 잘하고 있었기 때문에 조금 나태했다. 가만히 있어도 신기술을 적용한 모델이 쭉쭉 나오니까 딱히 신경 쓸 필요가 없었다.

**결과**:
- 개발에만 집중
- 회사가 원하는 방향과 조금 맞지 않게 됨
- 수동적 자세

### 2020년의 변화

#### 논문 읽기의 흥미 발견
**깨달음**:
- 논문을 안다 = 새로운 기술을 배운다
- 기술을 배울수록 남들보다 더 많이 알게 됨
- 능동적 학습의 시작

#### 회사 규모 확대와 새로운 기술 필요성
- **Notion**: 인원이 많은 구조에서 업무 관리 및 추적 용이
- 협업 최적의 조건 제공
- 새로운 기술 접목 시기 도래

#### 클라우드로의 방향 전환
- 도커가 직접적으로 사용되기 시작
- 클라우드 서비스 구조에 가까워짐
- 기존 시스템 활용 → 새로운 기술 도입

## Company vs Personal Growth

**질문들**:
- 회사의 성장 방향이란 뭘까?
- 여태까지 그 성장 방향을 우리에게 맡기지 않았을까?
- 회사의 성장 방향을 대답하자면 뭐라고 답해야 할지?

**고민**:
- 개인이 성장하면 회사가 성장하는 것 아닌가?
- 하지만 성장에도 **방향**이 있다
- 그 방향을 맞춰야 회사와 내가 같이 성장

### 나의 선택: 클라우드

**Why Cloud?**:
- 회사: 전문가적 지식을 가진 연구원 원함
- 나: 클라우드를 밀고 싶다

**갈등**:
- 회사와 직원이 원하는 방향이 다를 때가 많다
- 그래도 나는 클라우드를 선택

**이유**:
- 미래 기술 트렌드
- 실제 서비스 배포 경험 필요
- 확장 가능한 시스템 구축

### KPI vs OKR

#### KPI (Key Performance Indicator)의 한계
- 핵심 성과 **지표**
- 성과를 측정하는 도구
- 변화 측정 가능
- **하지만 목적이 빠져있음**

#### OKR의 도입
- **O**bjective and **K**ey **R**esults
- 목적과 핵심 성과
- 무조건 높으면 좋다 → 핵심 목표 추리기
- OKR 달성 = 회사 성장과 직접 연관

**3월부터 OKR 도입**:
- 주간 계획에 적용
- 목표 중심 업무 관리
- 더 명확한 방향성

## Infrastructure Transformation Insights

### Server PC (Truck) 도입 경험

**문제 상황**:
- GUI 지원 없는 Server OS 전용 PC
- 기존 Windows 환경과 완전히 다름
- 빠른 적응 필요

**해결 과정**:
- Ubuntu Server OS 선택
  - 가장 사용하기 쉬운 Server OS
- 예상보다 빠르게 세팅 완료

**파급 효과**:
- Linux가 속도 및 유연함에서 Windows보다 좋다는 것 입증
- 도커 도입 필요성 인식 확산
- **대부분 컴퓨터 Windows-Ubuntu 듀얼 부팅 전환**

**깨달음**:
- 변화는 어렵지만 필요하다
- 한 번의 성공 사례가 전체를 변화시킬 수 있음

### Docker의 혁명적 가치

#### 문제 인식
**Environment Hell**:
- 논문 코드 테스트 시 막대한 시간 소요
- 팀원 간 코드 공유 시 환경 차이 문제
- Virtual Machine은 무겁고 비효율적

#### Docker의 해결책
**Direct Benefits**:
- 환경 가상화로 설정 시간 단축
- 팀원 간 환경 통일
- 빠른 실험 및 테스트

**Unexpected Benefits**:
- 한 테스트당 한 개의 독립 환경
- 로컬 → 서버 전환 작업 불필요
- **코드 전달 문제 대부분 해소**

**깨달음**:
- 좋은 도구는 예상치 못한 부가 가치를 제공
- 인프라 투자는 장기적 생산성 향상

### Notion의 협업 혁신

**깨달음**:

- 투명성이 생산성을 높인다

## Key Takeaways

### 1. 논문의 가치
논문은 단순 지식이 아닌 **앞으로 무엇을 해야 할지 가이드**를 주는 힌트다.

### 2. 도구의 힘
좋은 도구(Docker, Notion)는 단순 편의가 아닌 **일하는 방식 자체를 변화**시킨다.

### 3. 문서화의 중요성
잘 작성된 문서는 **물리적 거리를 극복**하고 협업을 가능하게 한다.

### 4. 변화의 시작
한 사람의 성공 사례(Ubuntu Server)가 **전체 조직의 변화**(듀얼 부팅)를 만들 수 있다.

### 5. 방향의 중요성
개인 성장과 회사 성장의 **방향을 맞추는 것**이 진정한 성장이다.

### 6. 능동성
기다리지 말고 **직접 찾아서 배우기**. 논문 읽기, 새 기술 실험.

## Final Reflection

2020년은 깨어난 한 해였다. 2019년의 나태함에서 벗어나 능동적으로 학습하고, 현대적 도구를 도입하며, 클라우드라는 방향을 정했다.

**"나의 2020년은 성장 시작의 기점"**

이제 논문을 읽고, Docker를 쓰고, 클라우드를 향해 나아간다. 회사가 원하는 방향과 다를 수 있지만, 내가 믿는 길을 간다. 왜냐하면 **다양성이 조직을 강하게** 만들고, **각자의 전문성이 팀을 완성**하기 때문이다.
