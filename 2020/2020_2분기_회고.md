## Deep Manager 사이트 빌드

프로젝트 시작 전 - DM 프로젝트 시작 전 까지는 웹으로 제대로 구현되어있는 딥러닝 시연 사이트가 없었습니다. 대부분의 시연이 로컬에서 동작하는 프로그램을 들고다니며 시연해야 했고 제작된 언어가 c++, python 이였기 때문에 UI 부분에서 사용자가 편한 UI 를 제공하기 어려웠 습니다.

프로젝트 진행 - 웹을 통해 딥러닝 기술을 시연할 수 있는 서비스를 만들어야 한다고 했을때 주저없이 python 기반의 flask 를 채용했 습니다. 이 선택은 후에 가벼운 API 를 만들어 빠르게 배포할 수 있다는 점에서 후회하지 않는 선택이 되었고 flask, 웹 통신, 브라우져의 동작 등의 기술을 익히는데 도움이 되었습니다. 또한 서비스의 속도와 잦은 업데이트로 인한 서버점검상태를 방지하기 위해 도입한 도커 컨테이너와 컨테이너 병렬화 기술은 실제로 서비스를 병렬화하여 효율적으로 관리할 수 있도록 설계할 수 있는 능력을 얻을 수 있었습니다.

프로젝트 후기 - 비록 로그인, DB, 트래픽 최적화, 세션관리 등의 복잡한 기능 구현은 경험해 보지 못했지만 이 프로젝트를 통해 기초적인 API 설계방법, flask API 의 배포방법 등을 얻을 수 있었습니다. 이러한 경험을 바탕으로 다른 API 를 제작해 배포해야 한다고 했을때 GY 내부 인력만으로도 합리적인 기간 내에 배포할 수 있게 되었다고 생각합니다.

## Poldrone API 설계

얻게된 것 - 이 프로젝트는 이화트론과의 많은 협업이 필요한 프로젝트 였습니다. GY에서는 API 부분을 담당하여 진행하게 되었는데 서로 물리적으로 떨어져 있는 상황에서 문제없이 프로젝트를 설계하기 위해 GY 쪽에서 API 에 대한 통신방법과 규약 등을 문서화 하여 공유하였고 이러한 경험이 후에 API 를 제작하여 배포하는데 필요한 문서등을 구성하는데 도움이 될것으로 생각합니다
